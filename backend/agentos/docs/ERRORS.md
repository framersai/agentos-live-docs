**Note on User-Friendly Messages:**
If a specific user-friendly message is not listed, the system defaults to using the message provided when the `GMIError` was instantiated. If that message is also not available, it defaults to the generic message: "An unexpected error occurred."

**Note on HTTP Status Codes:**
The "Default HTTP Status Code" column reflects the status code determined by the `getHttpStatusCode` method in the `GMIError` class. If a specific code is passed during `GMIError` instantiation, that will take precedence for that specific error instance. The table shows the code derived from the `GMIErrorCode` as per the `statusCodeMap` or the default of `500` if not explicitly mapped.
